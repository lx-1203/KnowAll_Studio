# 知识点学习系统增强 - 需求文档

## 1. 功能概述

对 KnowAll_Studio 现有学习系统进行三大模块增强：
1. **知识点总结生成优化**：用户上传文件后，AI 生成完整、分级、结构化的知识点总结大全（Markdown 格式）
2. **多子 Agent 自动并行调度**：上传并生成知识点后，自动并行调用题库生成、实时答题、思维导图、外语学习、学习计划五个子 Agent
3. **记忆回流反馈系统**：持久化答题记录，建立知识点-题目-记忆卡全覆盖映射，实现薄弱环节自动检测与复习推送

---

## 2. 用户故事

### 2.1 知识点总结生成优化

**US-01**: 作为学生，我上传一份教材 PDF 后，希望系统自动生成一份完整的知识点总结，包含所有章节和具体知识点的详细解释，不要遗漏任何内容，以便我全面复习。

**US-02**: 作为学生，我希望知识点总结有清晰的层级结构（章→节→知识点），便于我按逻辑顺序学习，而不是零散的知识碎片。

**US-03**: 作为学生，我希望每个知识点不仅包含名称，还包含详细解释、关联概念和具体示例，帮助我深入理解和举一反三。

**US-04**: 作为学生，我希望知识点总结以 Markdown 格式输出，支持多级标题、列表嵌套、代码块、表格等丰富格式，方便我导出到其他笔记工具。

### 2.2 多子 Agent 自动并行调度

**US-05**: 作为学生，我在上传文档后，希望系统自动为我生成配套的练习题（选择题、填空题、简答题等），且每道题都能追溯到对应的知识点，省去我手动出题的时间。

**US-06**: 作为学生，我希望有一个交互式答题界面，可以逐题作答、即时查看对错和解析，帮助我检验学习效果。

**US-07**: 作为学生，我希望系统能将知识点自动转化为可视化的思维导图，方便我从宏观上把握知识体系结构。

**US-08**: 作为语言学习者，当我上传英语/日语等语言类学习材料时，希望系统能自动提取生词表（单词、音标、释义、例句、词性），辅助我同步积累词汇。

**US-09**: 作为学生，我希望系统能根据知识点总结，自动为我生成短期（1-3天按小时）和长期（1-4周按天）学习计划，并结合艾宾浩斯遗忘曲线安排复习节点，科学指导我的学习节奏。

### 2.3 记忆回流反馈系统

**US-10**: 作为学生，我希望我的答题记录（对错、耗时、重做情况）能被系统记住，以便了解自己的学习状态和薄弱点。

**US-11**: 作为学生，我希望每个知识点都有关联的记忆卡和练习题，系统能自动检测我哪些知识点掌握薄弱（正确率低于阈值），并针对性地推送复习内容。

**US-12**: 作为学生，我希望在系统生成题库后，能看到覆盖率报告（知识点总数 vs 已覆盖题目数），确保没有遗漏的知识点。

## 3. 验收标准

### 3.1 知识点总结生成

| 编号 | Given | When | Then |
|------|-------|------|------|
| AC-01 | 用户已上传一份文档 | 调用知识点总结生成 API | 返回完整的 Markdown 文档，包含多级标题（至少到三级） |
| AC-02 | 知识点总结已生成 | 查看输出内容 | 每个三级知识点包含：名称、详细解释（≥50字）、关联概念、示例 |
| AC-03 | 文档包含多个章节 | 查看生成结果 | 一级标题对应大章节，二级对应小节，三级对应具体知识点，层级逻辑正确 |
| AC-04 | 文档内容较多（>50页） | 调用生成 API | 不限制输出长度，完整覆盖文档所有知识点，无截断 |

### 3.2 多子 Agent 并行调度

| 编号 | Given | When | Then |
|------|-------|------|------|
| AC-05 | 知识点总结已生成 | 触发 Agent 调度 | 题库生成 Agent、实时答题 Agent、思维导图 Agent、学习计划 Agent 并行启动 |
| AC-06 | 知识点总结已生成 | 查看生成的题目 | 每种题型至少生成 5 道题，每道题标注对应的知识点 ID |
| AC-07 | 答题界面已加载 | 用户逐题作答 | 每题提交后即时显示对/错和答案解析 |
| AC-08 | 知识点总结已生成 | 查看思维导图 | 展示层级化的可视化思维导图，结构与知识点总结对应 |
| AC-09 | 上传文档为语言学习类材料 | 触发调度 | 外语学习 Agent 自动激活，生成生词表（单词、音标、释义、例句、词性） |
| AC-10 | 上传文档为非语言类材料 | 触发调度 | 外语学习 Agent 不激活，其余四个 Agent 正常运行 |
| AC-11 | 知识点总结已生成 | 查看学习计划 | 短期计划（1-3天按小时排程）和长期计划（1-4周按天排程）均已生成 |
| AC-12 | 学习计划已生成 | 查看复习节点 | 计划中包含基于艾宾浩斯遗忘曲线的复习节点（第1/2/4/7/15天） |

### 3.3 记忆回流反馈系统

| 编号 | Given | When | Then |
|------|------|------|------|
| AC-13 | 用户已完成答题 | 查看答题记录 | 正确/错误、每题耗时、错题重做情况均被持久化存储 |
| AC-14 | 知识点已生成记忆卡 | 查看记忆卡详情 | 每张卡记录复习次数、正确率、上次复习时间 |
| AC-15 | 存在正确率低于 70% 的记忆卡 | 系统执行回流检测 | 对应卡片和错题自动推送至复习队列 |
| AC-16 | 题库生成完成 | 查看覆盖率报告 | 报告显示知识点总数、已覆盖数、覆盖率百分比、未覆盖知识点列表 |
| AC-17 | 任意知识点存在 | 检查映射关系 | 每个知识点至少对应 1 张记忆卡 + 1 道基础题 |

## 4. 非功能性需求

### 4.1 性能
- 知识点总结生成：单次调用 120s 内完成（利用 LLM 流式输出）
- 多 Agent 并行调度：总耗时不超过最慢 Agent + 5s（调度开销）
- 记忆回流检测：单次扫描 < 5s（基于数据库查询）
- 覆盖率报告生成：< 2s

### 4.2 可靠性
- Agent 并行调用失败时不影响其他 Agent（独立错误隔离）
- 记忆卡与知识点映射支持重建（数据库级外键约束 + 级联更新）
- LLM 生成失败时支持自动重试（复用现有重试机制，max 3次）

### 4.3 可扩展性
- Agent 注册机制：新 Agent 可通过统一接口注册到调度器
- 题目类型可扩展（复用现有 9 种题型模板 + 新增模板注册）
- 复习算法可替换（FSRS / 艾宾浩斯 / 自定义间隔算法）

### 4.4 数据一致性
- 知识点 ID 作为全局唯一标识，贯穿题目、记忆卡、答题记录
- 删除知识点时级联标记关联题目和记忆卡为"待清理"
- 覆盖率报告实时查询，不使用缓存
