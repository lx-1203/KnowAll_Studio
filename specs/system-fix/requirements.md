# 系统功能完善 - 需求文档

## 1. 功能概述

对 KnowAll Studio（智识工坊）现有系统进行 11 项功能缺陷修复，涵盖数据完整性、算法准确性、资源管理和用户体验四个方面。目标是将系统从 MVP 级别提升到可靠可用的生产级别。

## 2. 用户故事

### 数据完整性

- **US-01**: 作为用户，我希望上传文档后返回的 chunk ID 是有效的，以便我能选择特定 chunk 进行知识树生成。
- **US-02**: 作为用户，我希望删除文档时所有关联数据（包括向量索引）都能被清理，以免占用磁盘空间。
- **US-07**: 作为用户，我运行全链路生成时不应产生重复的题目和闪卡，以免题库混乱。

### 算法准确性

- **US-03**: 作为用户，我希望所有文档 chunk 都能参与题目和闪卡生成，而不是只有前 5 个，以便充分利用全部学习资料。
- **US-04**: 作为用户，我希望填空题/简答题的评分标准合理严格，以便准确反映我的掌握程度。
- **US-06**: 作为用户，我希望 FSRS 记忆算法使用正确的参数计算复习间隔，以便科学安排复习计划。
- **US-09**: 作为用户，我希望生成的每道题都能追溯到具体来源段落，以便针对性复习。

### 资源管理

- **US-05**: 作为用户，我希望全链路生成包含知识大纲阶段（与 PipelineStage 枚举一致），以便获得完整的知识梳理。

### 系统可靠性

- **US-08**: 作为用户，我希望流式 AI 对话能正确保存对话记录，不会因为 session 关闭而丢失消息。
- **US-10**: 作为分享者，我希望所有分享类型都能正确验证资源是否存在，以便接收方不会遇到空数据。
- **US-11**: 作为用户，当我有大量数据时，列表接口应支持分页加载，以便页面快速响应。

## 3. 验收标准

### US-01: 上传返回有效 chunk ID

- **Given** 用户上传了一个文档
- **When** 上传解析完成
- **Then** 返回的 chunks 列表中每个 chunk 的 id 字段为有效的 UUID 字符串（非 None）

### US-02: 删除文档清理向量数据

- **Given** 用户已索引了一个文档到向量库
- **When** 用户删除该文档
- **Then** ChromaDB 中与该文档所有 chunk 关联的向量条目被移除

### US-03: 全部 chunk 参与生成

- **Given** 一个文档有 15 个 chunk
- **When** 用户使用全部 chunk 生成题目
- **Then** 所有 15 个 chunk 的内容被合并发送给 LLM（或按 token 预算分批处理）

### US-04: 严格的文本答案检查

- **Given** 一道简答题正确答案是 "细胞核是细胞的控制中心"
- **When** 用户回答 "的"
- **Then** 判定为错误（不再是子串匹配通过）

### US-05: Pipeline 包含大纲阶段

- **Given** 用户运行全链路
- **When** 知识树生成完成后
- **Then** 系统生成 Markdown 知识大纲，Pipeline 进度包含 outline 阶段

### US-06: FSRS 使用 w 参数

- **Given** 用户复习一张闪卡
- **When** FSRS 计算下次复习间隔
- **Then** 计算过程使用 DEFAULT_W 参数而不是硬编码魔术数字

### US-07: Pipeline 去重

- **Given** 用户已对某文档运行过一次全链路
- **When** 用户再次对同一文档运行全链路
- **Then** 系统提示已有生成结果或跳过重复生成

### US-08: 流式对话保存消息

- **Given** 用户正在进行 SSE 流式 AI 对话
- **When** 对话完成
- **Then** assistant 消息被正确保存到数据库

### US-09: 题目-段落溯源

- **Given** 系统从多个 chunk 生成题目
- **When** 保存题目时
- **Then** 每道题的 source_chunk_id 指向最佳匹配的 chunk（或标记为复合来源）

### US-10: 分享验证

- **Given** 用户尝试分享一个不存在的 question ID
- **When** 创建分享链接
- **Then** 收到 404 错误提示

### US-11: 列表分页

- **Given** 用户有 200+ 条文档记录
- **When** 访问文档列表页面
- **Then** 默认显示第一页（如 20 条），支持翻页加载更多

## 4. 非功能性需求

- **向后兼容**: 修复不应破坏现有已存储的数据
- **性能**: 分页后列表接口响应时间 < 500ms
- **LLM 成本**: 扩大 chunk 使用量时需要评估 token 预算，避免单次请求过大
- **安全性**: 分享验证必须严格执行
